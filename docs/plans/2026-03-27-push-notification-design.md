# 浇水提醒推送通知设计

**目标**：为护花使者 App 实现本地通知和服务器推送功能，当植物需要浇水时通过系统通知提醒用户。

**架构**：使用极光推送 (JPush) 实现国内用户推送通知，配合 APScheduler 定时任务检测到期提醒。

---

## 功能需求

1. **服务器推送**：后端定时检查到期提醒，通过极光推送发送通知
2. **本地通知**：App 可主动检查并显示本地通知
3. **通知跳转**：点击通知可跳转到 App 提醒页面

## 技术方案

### 后端 (Python/FastAPI)

- **定时任务**：使用 APScheduler 每6小时检查一次到期提醒
- **推送服务**：集成极光推送 SDK，当提醒到期时发送推送
- **数据模型**：Reminder 模型已有 `next_due` 字段，天然支持定时检查

### 前端 (React Native/Expo)

- **推送接收**：App 集成极光推送 SDK (jpush-react-native)
- **通知处理**：App 启动时注册通知，接收通知后跳转到对应页面
- **本地通知**：使用 expo-notifications 显示本地通知（备选）

## 数据流

```
1. 用户在 App 创建浇水提醒 → 后端存储提醒记录 (next_due)
2. APScheduler 定时任务每6小时执行
3. 查询 next_due <= 当前时间 的提醒
4. 调用极光推送 API 发送通知
5. 用户手机收到系统推送通知
6. 用户点击通知 → 打开 App 跳转提醒页面
```

## 通知内容示例

- **标题**：🌱 浇水提醒
- **内容**：您的"绿箩"今天需要浇水啦！
- **Extra**：{ "type": "reminder", "reminder_id": 123 }
