# 植物百科分类改进设计

## 背景

用户反馈：在养护百科页面中，点击分类（如"室内"、"多肉"等）后，每个分类显示的植物都是一样的，没有正确进行分类筛选。

## 目标

在百科页面中，点击分类后，在当前页面展示该分类下的植物列表，实现真正的分类筛选功能。

## 设计方案

### 1. 页面布局

```
┌─────────────────────────────┐
│  植物分类                   │
│  [全部] [室内] [多肉] [开花] │
└─────────────────────────────┘
         ↓ 点击分类后
┌─────────────────────────────┐
│  观叶植物 (共 15 种)        │
│  ┌────┐ ┌────┐ ┌────┐      │
│  │绿萝│ │虎皮│ │酒瓶│      │
│  └────┘ └────┘ └────┘      │
│  ┌────┐ ┌────┐ ┌────┐      │
│  │... │ │... │ │... │      │
│  └────┘ └────┘ └────┘      │
└─────────────────────────────┘
```

### 2. 交互逻辑

- **默认状态**：显示热门植物列表
- **选择分类后**：
  - 显示该分类的植物网格
  - 显示分类名称和植物总数
  - 切换分类时更新列表
- **点击植物卡片**：跳转到植物详情页

### 3. 组件设计

#### 分类植物区域
- 标题栏：显示分类名称 + 植物数量
- 网格布局：3列，卡片样式与热门植物一致
- 空状态：无数据时显示提示信息

### 4. 视觉设计

- 保持与热门植物一致的卡片样式
- 标题使用 sectionHeaderRow 样式
- 卡片使用 plantCard 样式

## 技术实现

- 使用现有的 `plants` 状态存储分类植物数据
- 点击分类时调用 `getPlants({ category })` 获取数据
- 在分类区域下方、条件渲染显示分类植物列表

## 验收标准

1. 点击分类后，页面正确显示该分类的植物
2. 分类名称和植物数量正确显示
3. 点击植物卡片可跳转到详情页
4. 切换分类时列表正确更新
