# 我的花园功能优化设计

**日期**: 2026-03-19

## 需求概述

优化"我的花园"页面，实现：
1. 功能增强：植物照片、智能提醒、AI 病虫害识别
2. 数据展示：统计概览、养护日历

## 页面结构

```
GardenScreen (我的花园)
├── Header - 标题 + 添加按钮
├── StatsBar - 统计概览卡片
│   ├── 植物总数
│   ├── 本月养护次数
│   └── 健康状态分布
├── CalendarStrip - 养护日历条
│   └── 本月日期 + 记录标记
├── PlantList - 植物卡片列表
│
PlantDetailScreen (植物详情)
├── Header - 植物图片（可拍照）+ 基本信息
├── AI诊断按钮 - 一键诊断
├── StatsBar - 生长/健康统计
├── CareTab - 养护记录
│   ├── Timeline - 时间轴
│   └── AddCare - 添加记录（支持拍照）
├── GrowthTab - 生长追踪
│   └── 照片展示
├── HealthTab - 健康记录
│   └── 诊断历史
└── Settings - 提醒设置
```

## 功能设计

### 1. 植物照片功能

- 照片上传到后端，存储在 `backend/uploads/plants/` 目录
- 使用场景：添加植物时拍照、养护记录时拍照、生长记录支持拍照

### 2. 智能提醒功能

- 提醒类型：浇水、施肥、换盆
- 基于用户养护记录自动计算下次提醒时间
- 用户可手动设置提醒频率

### 3. AI 病虫害识别

- 复用现有病虫害识别 API
- 在植物详情页添加"一键诊断"按钮
- 识别结果保存到健康记录

### 4. 统计概览

- 植物总数
- 本月养护次数（浇水、施肥等）
- 健康状态分布（健康/一般/生病）
- 各位置植物数量

### 5. 养护日历

- 月视图日历
- 每日养护记录标记
- 点击日期查看详细记录

## 后端 API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/plants/my/{id}/photo | 上传植物照片 |
| GET | /api/plants/my/{id}/photos | 获取植物照片列表 |
| GET | /api/plants/my/{id}/reminders | 获取提醒设置 |
| PUT | /api/plants/my/{id}/reminders | 更新提醒设置 |
| GET | /api/plants/my/stats | 获取花园统计 |
