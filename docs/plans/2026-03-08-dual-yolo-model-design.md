# 双模型植物识别与病虫害诊断架构设计

**日期**: 2026-03-08
**版本**: 1.0

## 1. 概述

本设计文档描述了护花使者应用中双模型架构的实现方案，通过两个独立的 YOLO 模型分别实现植物种类识别和病虫害诊断功能，支持服务器端和边缘端部署。

## 3. 功能设计

### 3.1 植物识别模型 (Plant YOLO)

| 项目 | 说明 |
|------|------|
| 模型类型 | YOLOv11 |
| 输入 | 植物照片 |
| 输出 | 植物种类 ID、名称、置信度 |
| 植物类别 | 室内绿植、开花植物、多肉、蔬菜/草本（约 50+ 种） |
| 数据集 | PlantNet、Oxford Flowers-102、补充自有数据 |

**植物类别列表（计划）**：

- **室内绿植**：绿萝、吊兰、虎皮兰、龟背竹、发财树、幸福树、橡皮树、琴叶榕、滴水观音、观音莲
- **开花植物**：月季、杜鹃、君子兰、蝴蝶兰、长寿花、蟹爪兰、栀子花、茉莉花、茶花、牡丹
- **多肉植物**：景天、仙人掌、玉露、生石花、熊童子、乙女心、桃蛋、法师、东云
- **蔬菜/草本**：薄荷、罗勒、辣椒、番茄、草莓、香菜、小葱、大蒜

### 3.2 病虫害识别模型 (Pest YOLO)

| 项目 | 说明 |
|------|------|
| 模型类型 | YOLOv11 |
| 输入 | 病虫害照片 |
| 输出 | 病虫害类型 ID、名称、置信度 |
| 病虫害类别 | 常见虫害、病害（约 30+ 种） |
| 数据集 | PlantVillage、Kaggle Plant Disease |

**病虫害类别列表（计划）**：

- **常见虫害**：蚜虫、红蜘蛛、介壳虫、粉虱、蓟马、小黑飞、蜗牛、鼻涕虫
- **真菌病害**：白粉病、黑斑病、炭疽病、叶斑病、锈病、灰霉病、根腐病
- **生理性病害**：黄叶、晒伤、冻害、肥害、药害

## 4. 数据流设计

### 4.1 服务器端识别流程

```
1. 用户上传图片
2. API 接收图片
3. 调用 PlantRecognizer.recognize() -> 植物种类
4. 调用 PestDiagnoser.diagnose() -> 病虫害类型
5. 返回识别结果
```

### 4.2 边缘端识别流程

```
1. 用户拍照
2. APP 加载本地模型
3. 本地推理返回结果
4. 可选：同步到服务器端
```

## 5. API 设计

### 5.1 植物识别 API

```
POST /api/recognition/plant
Request:
  - image: File (图片文件)

Response:
  {
    "id": "1",
    "name": "绿萝",
    "confidence": 0.95,
    "category": "室内绿植",
    "care_tips": "喜阴，避免直射，保持土壤湿润"
  }
```

### 5.2 病虫害诊断 API

```
POST /api/diagnosis/pest
Request:
  - image: File (图片文件)

Response:
  {
    "id": "1",
    "name": "蚜虫",
    "confidence": 0.88,
    "category": "虫害",
    "treatment": "使用吡虫啉喷洒",
    "severity": "medium"
  }
```

### 5.3 组合识别 API

```
POST /api/recognition/full
Request:
  - image: File (图片文件)

Response:
  {
    "plant": {
      "id": "1",
      "name": "绿萝",
      "confidence": 0.95
    },
    "pest": null,  // 或病虫害信息
    "disease": null  // 或病害信息
  }
```

## 6. 模型管理

### 6.1 模型存储结构

```
backend/
├── models/
│   ├── plant/
│   │   ├── plant_yolo11n.pt      # 植物识别模型
│   │   └── plant_classes.json    # 类别映射
│   └── pest/
│       ├── pest_yolo11n.pt       # 病虫害识别模型
│       └── pest_classes.json     # 类别映射
```

### 6.2 模型加载策略

- 服务器端：启动时加载两个模型到内存
- 边缘端：按需加载，支持模型更新和缓存

## 8. 部署配置

### 8.1 Docker Compose 配置

后端服务需要扩展：

```yaml
services:
  backend:
    volumes:
      - ./backend/models:/app/models
    environment:
      - PLANT_MODEL_PATH=/app/models/plant/plant_yolo11n.pt
      - PEST_MODEL_PATH=/app/models/pest/pest_yolo11n.pt
```

### 8.2 边缘端配置

- 模型量化（INT8）以减少体积
- 支持离线推理
