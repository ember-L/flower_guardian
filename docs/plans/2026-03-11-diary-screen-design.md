# 养花日记页面完善设计方案

**创建日期**: 2026-03-11
**功能模块**: 养花日记 (Plant Diary)

## 1. 需求概述

全面完善养花日记页面，实现以下目标：
- 连接后端 API，实现完整的 CRUD 功能
- 支持图片为主的日记（拍照 + 相册选择）
- 支持生长数据记录（高度、叶片数量）
- 实现生长曲线可视化
- 支持日记对比功能

## 2. 页面结构

```
养花日记模块
├── DiaryScreen          # 主页面 - 日记列表 + Tab 切换
├── WriteDiaryScreen    # 写日记页面
├── DiaryDetailScreen   # 日记详情 + 对比功能
└── GrowthCurveScreen   # 生长曲线页面
```

## 3. 页面详细设计

### 3.1 主页面 (DiaryScreen)

#### 布局结构
- **SafeAreaView**:刘海 适配屏
- **Header**: 标题 + 写日记按钮
- **TabBar**: "我的日记" | "生长记录"
- **Content**: 列表内容

#### UI 规格
| 元素 | 样式 |
|------|------|
| 标题 | 20px, 粗体, 主标题色 |
| 副标题 | 14px, 次级文字色 |
| 写日记按钮 | 主色背景, 圆角 12px, 白色文字 |
| Tab | 未选中: 次级文字, 选中: 主色背景 + 白色文字 |
| 日记卡片 | 白色背景, 圆角 16px, 轻微阴影 |

#### 日记卡片内容
```
┌─────────────────────────────────┐
│ [头像] 绿萝        [有新变化]   │
│        2024-01-20              │
│ ┌─────┐ ┌─────┐ ┌─────┐ ┌────┐│
│ │图片1│ │图片2│ │图片3│ │+6  ││
│ └─────┘ └─────┘ └─────┘ └────┘│
│ 今天给绿萝换了一个大一点的花盆.. │
│                              │
│ ❤️ 12   💬 3                  │
└─────────────────────────────────┘
```

#### Tab 2: 生长记录
- 显示每个植物的生长数据汇总
- 点击进入 GrowthCurveScreen 查看详情

### 3.2 写日记页面 (WriteDiaryScreen)

#### 布局结构
- **Header**: "写日记" + 取消/保存按钮
- **植物选择**: 下拉选择我的植物
- **图片区域**: 图片网格 + 添加按钮
- **生长数据**: 高度、叶片数输入（可选）
- **文字输入**: 多行文本框

#### 功能规格
| 功能 | 规格 |
|------|------|
| 图片选择 | 拍照 + 相册, 最多 9 张 |
| 图片预览 | 网格布局, 3列, 可删除 |
| 植物选择 | 从用户植物列表选择 |
| 生长数据 | 高度 (cm): 数字输入, 叶片数: 数字输入 |
| 文字记录 | 最大 500 字 |

#### 交互流程
1. 选择植物
2. 添加图片（可选）
3. 填写生长数据（可选）
4. 填写文字描述（可选）
5. 保存日记

### 3.3 日记详情页面 (DiaryDetailScreen)

#### 功能
- 查看日记完整内容
- 图片大图预览（可滑动）
- 编辑日记
- 删除日记
- 对比功能：与之前的日记对比

#### 对比功能设计
- 显示"对比上次"按钮
- 点击后展示：当前日记图片 | 上次日记图片
- 支持左右/上下布局切换

### 3.4 生长曲线页面 (GrowthCurveScreen)

#### 布局结构
- **顶部**: 植物选择 + 时间范围选择
- **图表区**: 高度折线图 + 叶片数柱状图
- **数据列表**: 所有记录的数据点

#### 图表规格
| 图表 | 类型 | 数据 |
|------|------|------|
| 高度变化 | 折线图 | Y: cm, X: 日期 |
| 叶片数量 | 柱状图 | Y: 数量, X: 日期 |

#### 时间范围
- 最近 1 个月
- 最近 3 个月
- 最近 6 个月
- 全部

## 4. API 设计

### 现有 API（需确认已实现）
```
GET    /api/diaries              # 获取日记列表
POST   /api/diaries              # 创建日记
GET    /api/diaries/{id}        # 获取日记详情
PUT    /api/diaries/{id}        # 更新日记
DELETE /api/diaries/{id}        # 删除日记
```

### 新增 API（可选，如后端未实现可后续添加）
```
GET /api/diaries/growth-data?plant_id={id}  # 获取生长数据（用于图表）
```

### Diary Schema（后端已有）
```python
class DiaryCreate:
    user_plant_id: int      # 必填
    content: str            # 可选
    images: List[str]       # 可选，图片URL列表
    height: Optional[int]   # 可选，高度cm
    leaf_count: Optional[int]  # 可选，叶片数量
```

## 5. 技术实现要点

### 5.1 依赖库
- `react-native-image-picker`: 图片选择
- `react-native-svg` + `react-native-svg-charts` 或 `victory-native`: 图表
- `axios`: API 请求

### 5.2 数据流
```
用户操作 → React State → API Call → 后端 → 数据库
                    ↓
              更新本地缓存/状态
```

### 5.3 错误处理
- 网络错误: 显示 Toast 提示，支持重试
- 图片上传失败: 显示失败标记，支持重新上传
- 数据验证: 表单级别验证，实时反馈
