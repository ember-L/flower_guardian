# 诊断页面功能完善设计方案

**日期**: 2026-03-14

**目标**: 完善病症诊断页面的功能，包括AI问诊、养护知识、病害识别优化

---

## 1. 概述

本文档描述护花使者APP诊断模块的改进方案，主要包括：
1. 问诊室 - AI对话问诊功能
2. 养护知识 - 个性化+通用知识
3. 病害识别 - 对接后端API + 离线识别

---

## 2. 页面结构

```
诊断页面 (DiagnosisScreen)
├── 头部区域
│   ├── 页面标题
│   └── 状态指示器（在线/离线）
├── Tab/快速入口区域
│   ├── 病害识别入口
│   ├── 问诊室入口
│   └── 养护知识入口
└── 诊断历史入口
```

---

## 3. 功能详细设计

### 3.1 问诊室 (AI问诊)

#### 功能定位
- 对接阿里云DashScope Qwen API
- 通过后端代理调用（安全）
- 支持文字+图片输入
- 保存对话历史，支持继续对话

#### 页面结构
```
问诊室列表页 (ConsultationListScreen)
├── 顶部导航
│   ├── 返回按钮
│   └── 新建对话按钮
└── 对话历史列表
    └── 点击进入对话详情

问诊对话页 (ConsultationScreen)
├── 顶部导航
│   ├── 返回按钮
│   └── 标题（问诊室）
├── 消息列表
│   ├── 用户消息气泡（右侧）
│   └── AI消息气泡（左侧）
└── 输入区域
    ├── 图片附件按钮
    ├── 文字输入框
    └── 发送按钮
```

#### API设计

**后端代理接口 (新增)**
```
POST /api/ai/chat
Body: {
  messages: [
    { role: "user", content: "..." },
    { role: "assistant", content: "..." }
  ],
  context?: {
    diagnosis_history: [...],
    plant_type?: string
  }
}
Response: {
  message: { role: "assistant", content: "..." }
}
```

**前端存储**
- 使用 AsyncStorage 保存对话历史
- 结构: `{ conversations: [{ id, title, messages: [], updated_at }] }`

#### 对话上下文
- 优先使用诊断历史作为上下文
- 无诊断历史时，使用通用植物知识

---

### 3.2 养护知识

#### 功能定位
- 预置通用植物养护知识
- 基于诊断历史生成个性化建议

#### 页面结构
```
养护知识页 (KnowledgeScreen)
├── 顶部Tab
│   ├── 通用知识
│   └── 个性化建议（如有诊断历史）
└── 内容区域
    ├── 知识文章列表
    └── 文章详情
```

#### 内容来源

**通用知识** - 预置内容（静态）
- 浇水技巧
- 光照要求
- 施肥指南
- 病虫害预防
- 季节养护

**个性化建议** - 动态生成
- 根据诊断历史中的病害/虫害
- 结合当前季节
- 生成针对性的养护建议

---

### 3.3 病害识别

#### 功能定位
- 对接后端真实API
- 提供离线识别功能
- 参考植物识别的混合模式

#### 识别流程

```
用户拍照/选择图片
    ↓
检查网络状态
    ↓
┌── 在线模式 ──→ 调用 /api/diagnosis/full
│   (有网络)        ↓
│              返回识别结果 + 建议
│                   ↓
└── 离线模式 ──→ 使用本地ONNX模型
    (无网络)        ↓
              返回基础识别结果
```

#### API对接

**后端接口**
```
POST /api/diagnosis/full (需认证)
POST /api/diagnosis/pest (需认证)
```

**APP端修改**
- 更新 hybridRecognition 服务
- 对接真实的 `/api/diagnosis/full` API
- 参考 PlantRecognition 的在线/离线切换逻辑

---

### 3.4 诊断历史

现有功能保持不变：
- 列表展示历史诊断记录
- 支持收藏
- 支持再次诊断

## 4. UI/UX 设计

### 4.1 配色方案

保持现有主题：
- 主色: #f46 (花瓣粉)
- 辅色: #52c41a (清新绿)
- 强调色: #faad14 (暖黄)

### 4.2 问诊室对话UI

**消息气泡**
- 用户消息: 粉色背景 (#f46)，白色文字，右侧
- AI消息: 白色背景，灰色文字，左侧
- 带图片: 图片圆角显示，下方文字

**输入框**
- 底部固定
- 左侧: 图片附件按钮
- 中间: 文字输入，多行自适应
- 右侧: 发送按钮

### 4.3 养护知识UI

**列表页**
- 卡片式布局
- 图标 + 标题 + 简介

**详情页**
- 标题 + 正文
- 必要时可配图
