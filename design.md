产品名称： 护花使者
Slogan建议：
主标语： 你的掌上植物管家，让养花不再凭感觉。
副标语： 养花小白进阶指南，从此告别“植物杀手”。
目标用户：
核心用户： 城市租房党、办公室白领（生活枯燥想养绿植，但没经验、没时间）。
潜力用户： 园艺爱好者、退休老人（侧重记录与分享）。
核心价值： 降低养花门槛，通过工具+内容的模式，建立用户信任感。

技术栈：
后端：Fast API + PostgesQL
前端：React Native + TailwindCSS（APP+Web）
CV模型：Yolov11进行训练 
大语言模型：QWen API

二、 核心功能细化
基于你提出的功能，我们需要细化其交互逻辑，确保“操作轻松”：

1. 花卉识别（入口功能）
功能点： 拍照识别、相册导入。
亮点设计：
一键生成“档案卡”： 识别成功后，不只显示名字，直接弹窗询问“是否加入我的花园？”。
相似种对比： 对于难以辨别的品种（如月季vs玫瑰），提供对比图，让用户二次确认，提高准确性。
2. 养护百科（内容核心）
痛点解决： 百度出来的攻略太复杂、术语多。
设计策略： “傻瓜式”卡片。
将养护难度分级（如：⭐⭐⭐⭐⭐ 难度等级）。
关键指标可视化：用图标代替文字（喜阳/耐阴、喜湿/耐旱、耐寒程度）。
避坑指南： 每个词条下设“常见死法”板块，直白告诉小白：这花千万别多浇水！
3. 智能提醒（留存关键）
功能点： 浇水、施肥、换盆、修剪。
亮点设计：
环境校准： 提醒不是死板的。用户初次添加植物时，询问“摆放位置”（南阳台/北卧室/办公室），系统根据环境自动调整浇水频率。
懒人模式： 如果用户点击“延期提醒”，系统会记录并学习用户的懒惰习惯，自动调整下次提醒时间。
4. 病症诊断（增值服务）
功能点： 拍照识别病虫害。
交互设计：
AI预诊： 识别黄叶、斑点、萎蔫，给出初步判断（如：疑似缺铁/疑似晒伤）。
专家/社区求助： AI无法判断时，一键发布到社区“急诊室”，由资深花友或专家人工解答（增加社区活跃度）。
5. 新手推荐（流量入口）
场景化推荐：
不直接列植物清单，而是做场景问答。
例如：“你家里光照好吗？” -> “经常出差吗？” -> “家里有猫吗？”（排除有毒植物）。
结果推荐：“养不死榜单”（如绿萝、虎皮兰、仙人掌），并附带“成活率承诺”。
6. 养花日记（社区与情感）
功能点： 成长记录、打卡、分享。
亮点设计：
对比图功能： 支持“同角度、同位置”的照片叠加对比，生成延时摄影般的成长GIF，极大增加用户成就感。
植物生命线： 自动生成植物的生长曲线图（记录高度、叶片数量）。
三、 UI/UX 设计建议
为了符合“操作轻松、内容实用”的初衷，设计风格建议如下：

主色调： 清新绿+ 暖木色/米黄色（Warm Beige）。代表生机与自然，视觉上不疲劳。
导航结构：
底部导航栏（4Tab）：
识别（首页）： 中间的大按钮，核心入口。
花园（我的植物）： 展示用户添加的植物列表，也是提醒中心。
百科/发现： 浏览内容、新手推荐。
我的： 个人中心、日记入口。
交互风格： 卡片式设计，大字体，少文字，多图标。减少层级跳转，尽量在一个页面完成操作。
